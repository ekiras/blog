Points To Remember


  • In this pattern the class always contains a Builder class that will create the object for this class.
  • This class can optionally have a private Constructor, this enforces the use of the Builder for making objects.
  • You should use this patter only for classes with many fields at least more than 4 to 6, and needs many overloaded constructors.
  • Avoid using this pattern when your class has less than 4 fields.
