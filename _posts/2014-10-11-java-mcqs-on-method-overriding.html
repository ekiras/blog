What happens if we override static methods of a class.
Compile time error Run time error
No error is thrown None of these
C
Static methods are class level methods and are not bound to the objects, thus static methods cannot be overloaded. If we try to overload static methods, both the methods will work and can be called with the reference of the class. We wont get any sort of error.

class A{
static void t(){}
}

class B extends A{

public void t(){}

}
Compile Time Error Runtime Error
Code works fine None Of these
A
Here method "t" of class B tries to override the method "t" of class A, but we cannot override a static method. Thus we will get a compile time error. When we will do A.t() the jvm will not be able to decide which method to call, the method of class A or method of class B, since both the mothods can be called with this syntax.

 void test(){

 public static void main(String args[]){
  A object = new B();
  object.test(); 
 }
}
Compile Time Error Runtime Error
Class A Class B
A
The code will result into compile time error because it fails the condition of overriding that the access specifier of the overriding method cannot be weaker than the actual method. In our example, method test() has public access and in the class B we are tring to assign a weaker access to this method that is default, thus it will give a compile time error.

The method test() gets overridden and hence object.test() will print "Class B". Variables of a class never gets overridden, thus object.var will give 10, the value of the variable in class A, because the reference of the object is of class A.
